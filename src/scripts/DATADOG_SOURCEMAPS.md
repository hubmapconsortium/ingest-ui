Datadog sourcemaps upload and version tagging

Overview

This project uses Datadog's `datadog-ci` to upload JavaScript sourcemaps generated by the production build so Datadog can unminify stack traces for RUM / Error Tracking and Logs.

Quick scripts (added to `package.json`)

- `npm run build:sourcemaps` — builds the app with production sourcemaps enabled.
- `npm run upload-sourcemaps` — runs a helper script that detects a release version and calls `datadog-ci sourcemaps upload`.
- `npm run upload-sourcemaps:dry-run` — same as above but performs a dry run.

Required environment variables

- `DATADOG_API_KEY` — required to upload sourcemaps (CI secret). If missing the upload script will fail (unless `--dry-run`).
- `DATADOG_SITE` — optional; defaults to `datadoghq.com` (use `datadoghq.eu` for EU).
- `DD_SERVICE` — optional; defaults to the `package.json` name or `ingest-ui`.
- `DD_ENV` — optional; environment name, defaults to `production`.
- `DD_VERSION` — optional; release version to attach to uploaded sourcemaps. If not provided the script uses `git rev-parse --short HEAD` or falls back to `package.json` version.
- `DD_MINIFIED_PATH_PREFIX` — optional; set to the URL path prefix where minified assets are served (defaults to `/static/js`).

CI example (GitHub Actions)

- Set `DATADOG_API_KEY` as a secret in the CI provider.
- In a deploy job run:

```bash
npm ci
npm run build:sourcemaps
DD_VERSION=$(git rev-parse --short HEAD) DD_SERVICE=ingest-ui DD_ENV=production npm run upload-sourcemaps
```

Notes

- Ensure the `--minified-path-prefix` matches the path shown in browser error stack frame URLs (e.g. `/static/js` or a full CDN URL). Incorrect prefix may prevent mapping.
- The helper script is `scripts/upload-sourcemaps.js` and uses `npx datadog-ci` under the hood. `datadog-ci` is added as a dev dependency.
- If you prefer to call `datadog-ci` directly in CI, the equivalent command is:
